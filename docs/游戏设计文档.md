# 工程伦理密室逃脱游戏 - 设计文档

## 一、项目概述

### 1.1 项目背景
本作品为《工程伦理》课程大作业，以网页密室逃脱游戏的形式，展示课程全部7章的知识点。玩家扮演一名被困在虚拟"科技伦理实验室"中的工程师，需要通过答题、解谜、情景判断等方式逐一解锁7个密室，最终逃出实验室。

### 1.2 技术栈
| 技术 | 版本 | 用途 |
|------|------|------|
| React | 19.2.0 | UI框架 |
| Vite | 7.2.4 | 构建工具 |
| TypeScript | 5.9.3 | 类型安全 |
| @vitejs/plugin-react-swc | 4.2.2 | SWC编译 + Fast Refresh |
| ESLint | 9.39.1 | 代码规范检查 |
| Tailwind CSS | 4.2.2 | 全局设计令牌 + 像素复古主题样式 |
| styled-components | 6.4.0 | 组件级样式封装（页面和组件） |
| react-router | 7.14.1 | 页面路由 |
| motion | 12.38.0 | 动画库 |
| React Context + useReducer | — | 状态管理（计划中） |
| 原生 HTML5 Drag & Drop | — | 拖拽交互（计划中） |
| localStorage | — | 进度存档（计划中） |

运行方式：
```bash
npm run dev      # 启动开发服务器
npm run build    # 构建生产版本（tsc -b && vite build）
npm run lint     # ESLint检查
npm run preview  # 预览生产构建
```

### 1.3 核心设计理念
- **寓教于乐**：将知识点融入游戏叙事，答题即解谜
- **沉浸体验**：像素复古 (8-bit) 视觉风格 + CRT 扫描线，完整的剧情线
- **多题型覆盖**：排序、分类、选择、填空、情景判断、密码锁
- **渐进解锁**：7个密室依次开放，完成前一章才能进入下一章

## 二、游戏流程设计

### 2.1 整体流程

```
开始界面 → 密室1 → 密室2 → 密室3 → 密室4 → 密室5 → 密室6 → 密室7 → 逃脱成功
         ↕        ↕        ↕        ↕        ↕        ↕        ↕
      (4个谜题) (4个谜题) (3个谜题) (2个谜题) (3个谜题) (4个谜题) (4个谜题)
```

### 2.2 单个密室流程

```
进入密室 → 密室介绍(叙事) → 房间探索场景（自由走动）
                                  │
                    ┌─────────────┼─────────────┐
                    ↓             ↓             ↓
               答题点A        答题点B   ...  答题点N
             （点击→角色走过去→弹窗答题）
                    │             │             │
                    └─────────────┼─────────────┘
                                  ↓
                       全部答题点完成 → 密室通关 → 下一密室
```

**探索机制说明：**
- 每个密室是一个2D俯视角场景，包含若干个答题点（发光标记）
- 玩家控制一个角色在房间内移动（点击目的地，角色自动寻路走过去）
- 点击答题点 → 角色走到答题点旁 → 弹出答题弹窗
- 答题点状态：未激活（发光脉冲）→ 进行中（角色走近）→ 已完成（绿色✓）→ 已锁定（灰色，需按顺序）
- 答题支持两种模式：顺序解锁（按编号依次解锁）或自由选择（可任意点击）
- 提示系统和重试机制在答题弹窗内触发

### 2.3 分数系统

| 项目 | 规则 |
|------|------|
| 每题满分 | 10~15分（按难度） |
| 提示扣分 | 使用提示扣该题50%分数 |
| 重试扣分 | 每次重试扣该题20%分数 |
| 总分上限 | 约280分（24题 × 平均11.7分） |
| 评级标准 | S级(>90%) / A级(>75%) / B级(>60%) / C级(>40%) |

### 2.4 成就系统

| 成就名 | 解锁条件 |
|--------|----------|
| 初出茅庐 | 通关第1章 |
| 伦理学者 | 通关第2章 |
| 职场达人 | 通关第3章 |
| 工匠之心 | 通关第4章 |
| 思维先锋 | 通关第5章 |
| 安全卫士 | 通关第6章 |
| 链上智者 | 通关第7章 |
| 完美逃脱 | 全程不使用提示通关 |
| 速通达人 | 30分钟内通关全部7章 |
| 伦理大师 | 总分超过90% |

## 三、7个密室详细设计

### 密室1：时间档案馆

**房间主题**：昏暗的档案室，墙壁上覆盖着发光的时间线和古老计算机蓝图。
**叙事引入**："你进入一个布满计算机历史设备的昏暗档案室。一个数字声音说：'要打开第一扇门，你必须追溯计算的起源。'"

| 编号 | 谜题名称 | 交互类型 | 知识点 | 分值 |
|------|----------|----------|--------|------|
| 1-1 | 计算史时间线 | 拖拽排序 | 图灵→ENIAC→深蓝→大数据→AlphaGo→ChatGPT | 10 |
| 1-2 | 计算机之父 | 单选题 | 图灵与冯·诺依曼的贡献辨析 | 10 |
| 1-3 | 数据单位排序 | 拖拽排序 | Byte→KB→MB→GB→TB→PB→EB→ZB | 10 |
| 1-4 | 大数据3V特征 | 多选题 | Volume/Velocity/Variety | 10 |

**谜题1-1详细内容：时间线排序**

叙事："一个损坏的时间线显示屏闪烁着。恢复事件的正确顺序。"

将6个事件按时间排列：
1. 图灵提出图灵机概念 (1936)
2. ENIAC诞生 (1946)
3. 卡斯帕罗夫 vs 深蓝 (1997)
4. Google日处理数据超24PB（大数据时代）
5. AlphaGo击败李世石 (2016)
6. ChatGPT发布 (2022)

**谜题1-2详细内容：计算机之父**

叙事："一个锁定的展示柜上提问：'谁被公认为计算机之父？'"

题目："谁是公认的'计算机之父'，他的核心贡献是什么？"
- A. 冯·诺依曼 —— 提出存储程序计算机体系结构（部分正确）
- B. 图灵 —— 提出图灵机和图灵测试，奠定计算理论基础 ✅
- C. 比尔·盖茨 —— 创立微软，推动个人电脑普及
- D. 乔布斯 —— 创造iPhone，引领移动互联网时代

解析：图灵被称为"计算机之父"，冯·诺依曼被称为"现代计算机之父"。

**谜题1-3详细内容：数据单位排序**

叙事："一个数据存储终端需要校准。将数据单位从小到大排序。"

正确顺序：Byte < KB < MB < GB < TB < PB < EB < ZB

**谜题1-4详细内容：大数据3V特征**

叙事："一扇上锁的门有一个扫描仪。它要求：'识别大数据的3V特征。'"

题目："大数据的3V特征包括哪些？"（多选）
- A. Volume（大量）✅
- B. Velocity（高速）✅
- C. Variety（多样）✅
- D. Vulnerability（脆弱性）
- E. Veracity（真实性）

## 四、房间探索系统设计

### 4.1 房间场景

每个密室是一个2D俯视角房间场景，以CSS/Canvas绘制，包含以下元素：

**房间地图：**
- 固定尺寸的场景区域（如 960×640），居中显示，超出部分可滚动
- 背景为该密室主题的房间平面图（用CSS渐变+SVG装饰元素绘制，不用图片）
- 房间内有家具/设备装饰物（纯装饰，不可交互，作为视觉遮挡和路径参考）
- 地面有微弱的网格线（#1e293b 低透明度），增强空间感

**答题点（PuzzlePoint）：**
- 每个谜题对应房间内的一个答题点，标记为一个发光圆形图标（直径约40px）
- 答题点样式：
  - 未激活：#00f0ff 脉冲发光 + 浮动动画（上下微动）+ 编号标签 "1-1"
  - 可交互：鼠标悬停放大 1.2x + 光圈扩散效果
  - 已锁定：#334155 暗灰色 + 锁图标 🔒（需先完成前置谜题）
  - 已完成：#22c55e 绿色 + ✓ 图标，不再可交互
- 答题点分布在房间的不同位置，与谜题主题相关（如"时间线排序"答题点靠近墙上的时间线屏幕）

**出口门：**
- 房间一侧有一扇门（视觉元素）
- 全部谜题完成后：门从 #334155 变为 #00f0ff 发光 + 开启动画 → 点击进入下一密室
- 未完成时：门为锁定状态，点击提示"还有 N 个谜题未完成"

### 4.2 角色

**角色外观：**
- 简约风格的小人（约 32×48px），俯视角，不需要精细立绘
- 用CSS绘制：圆形头部 + 矩形身体 + 简单四肢轮廓，#f8fafc 白色主色
- 身体周围有一圈 #00f0ff 的微弱光晕（标识玩家位置）
- 根据移动方向有4个朝向（上/下/左/右），用CSS transform翻转实现

**移动机制：**
- 点击房间空白处：角色以直线走到点击位置（避障不做要求，简单直线移动即可）
- 点击答题点：角色走到答题点旁边（答题点旁边预设一个"站位坐标"），到达后弹出答题弹窗
- 移动速度：约 150px/秒，带有走路帧动画（简单左右摆动模拟走路）
- 移动中不可触发其他交互，到达目的地后才可操作

**角色动画状态：**
| 状态 | 表现 |
|------|------|
| 站立 | 微微呼吸式上下浮动（2px） |
| 行走 | 左右交替晃动 + 位移 |
| 答题中 | 面朝答题点，静止 |
| 答对 | 头顶弹出 "+10" 绿色文字上浮消失 |
| 答错 | 身体短暂红色闪烁 |

### 4.3 答题弹窗

点击答题点且角色到达后，弹出一个模态弹窗：

**弹窗设计：**
- 全屏半透明黑色遮罩 (#0a0e1a, 70% opacity)
- 居中卡片：#111827 背景，#00f0ff 发光边框，圆角 16px，最大宽度 560px
- 卡片顶部：谜题编号 "谜题 1-1" + 叙事文本（#cbd5e1 斜体）
- 卡片中部：具体交互组件（排序/分类/选择/填空/密码锁/情景判断）
- 卡片底部：[💡 提示] [提交答案] 按钮

**答题后反馈：**
- 正确：弹窗变 #22c55e 边框 → 显示解析 → [下一题] / [返回房间]
- 错误：弹窗抖动 → 显示错误提示 → [重试] / [使用提示]
- 关闭弹窗：角色留在原地，答题点变为已完成状态

### 4.4 六种交互类型

| 组件 | 描述 | 使用场景 |
|------|------|----------|
| MultipleChoice | 单选/多选题，选项高亮选中状态 | 知识点直接考查 |
| DragSort | 拖拽排序（支持点击上下移动） | 时间线排序、单位排序、CDIO排序 |
| DragClassify | 拖拽到目标分类框（支持点击选择） | 概念分类、威胁分类 |
| PasswordLock | 数字/文字密码输入框 | 密码锁谜题 |
| ScenarioCard | 情景卡片 + 复选框 + 答后解析 | 情景判断题 |
| FillBlank | 文本中的空白填入 | 填空题 |

### 4.5 通用答题流程

答题弹窗内的通用流程（PuzzleModal）：
1. 弹窗打开 → 显示叙事文本（介绍场景）+ 谜题标题
2. 渲染具体交互组件（六种类型之一）
3. 提交按钮 → 判断正误
4. 正确：弹窗变绿色边框 + 显示解析 → [返回房间] 关闭弹窗
5. 错误：弹窗抖动 + 显示错误提示 → [重试] 或 [使用提示]

### 4.6 移动端适配
- 房间场景：双指缩放查看全貌，单击移动/交互
- 角色：点击屏幕任意位置移动（与PC端一致）
- 答题弹窗：全屏展示，不做缩放
- 拖拽排序：提供"点击选中 + 点击目标位置"的替代方式
- 拖拽分类：提供"点击选中 + 点击分类框"的替代方式

## 五、视觉设计方案

### 5.1 视觉风格

**像素复古风 (Pixel Retro)** — 8-bit 游戏美学 + CRT 显示器质感。
- 所有阴影使用实色偏移 `box-shadow: Npx Npx 0 0 #000`，不使用模糊阴影
- 动画使用 `steps()` 步进式，模拟低帧率画面
- 禁止字体平滑 (`-webkit-font-smoothing: none`)，保持像素锐利边缘
- 字体：Press Start 2P（标题）/ VT323（正文）/ Silkscreen（辅助）
- 页面级组件使用 styled-components 封装样式，通用动画类保留在 index.css

### 5.2 色彩体系

设计令牌定义在 `index.css` 的 `@theme {}` 中，详见 `docs/风格指南.md`。

```
文字色：#f4f4f4（主）、#94b0c2（次）、#566c86（弱）
```

## 六、状态管理设计

### 6.1 核心状态结构

```typescript
GameState {
  screen: 'start' | 'playing' | 'victory'  // 当前屏幕
  currentRoom: number                       // 当前密室(1-7)
  roomPhase: 'entrance' | 'exploring' | 'puzzling' | 'cleared'  // 密室阶段
  totalScore: number                        // 总分
  roomScores: Record<number, number>        // 每个密室得分
  puzzleStates: Record<string, PuzzleState> // 每个谜题状态
  characterPos: { x: number, y: number }    // 角色当前坐标
  activePuzzle: string | null               // 当前打开的答题弹窗ID
  unlockedRooms: number[]                   // 已解锁密室
  achievements: string[]                    // 已解锁成就
  startTime: number                         // 开始时间
  hintsUsed: number                         // 使用提示次数
}
```

### 6.2 状态流转

```
START_GAME → screen='playing', currentRoom=1, roomPhase='entrance'
ENTER_ROOM → roomPhase='exploring', 角色出现在入口位置
CLICK_PUZZLE_POINT → 角色移动到答题点旁, roomPhase='puzzling'
OPEN_PUZZLE_MODAL → activePuzzle=id
SOLVE_PUZZLE → puzzleStates[id]='solved', 更新分数, activePuzzle=null, roomPhase='exploring'
ALL_SOLVED → 出口门解锁
EXIT_ROOM → 解锁下一密室, roomPhase='cleared'
COMPLETE_GAME → screen='victory'
```

### 6.3 存档机制
- 每次状态变更自动保存到 localStorage
- 返回游戏时提供"继续游戏"和"新游戏"选项
- 存档key: `escape-room-ethics-save`
