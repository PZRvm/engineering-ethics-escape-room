# 工程伦理密室逃脱 - 视觉风格指南

## 一、设计理念

**像素复古风 (Pixel Retro Style)** — 以 8-bit 游戏美学为核心，融合 CRT 显示器质感，营造复古街机密室逃脱的沉浸体验。

核心原则：
- **锐利边缘**：禁止字体平滑，保持像素的锐利方块感
- **步进动画**：使用 `steps()` 而非平滑过渡，模拟低帧率游戏画面
- **纯色方块**：阴影用实色偏移（`box-shadow: Npx Npx 0 0 #000`），不使用模糊阴影
- **确定性动画**：背景粒子使用固定伪随机位置，避免每次渲染闪烁

## 二、字体体系

| 用途 | 字体 | CSS 变量 | 说明 |
|------|------|----------|------|
| 标题/按钮 | Press Start 2P | `--font-display` | 像素方块字体，用于主标题、按钮文字 |
| 正文/叙事 | VT323 | `--font-body` | 终端风格等宽字体，用于叙事文本、选项描述 |
| 辅助/装饰 | Silkscreen | `--font-mono` | 细腻像素字体，用于分数、时间、提示文字 |

全局设置：
```css
body {
  font-family: var(--font-body);
  font-size: 18px;
  -webkit-font-smoothing: none;       /* 禁止字体平滑 */
  -moz-osx-font-smoothing: grayscale;
  image-rendering: pixelated;          /* 保持像素锐利 */
}
```

## 四、组件规范

### 4.1 按钮 `.btn-neon`

```
┌────────────────────────┐
│  ▶ 即将开始             │  ← Press Start 2P, 0.7rem, uppercase
└────────────────────────┘
   4px 实色边框 (#41a6f6)
   4px 4px 0 0 #000 偏移阴影
```

状态：
| 状态 | 效果 |
|------|------|
| 默认 | `bg-card` 背景 + `pixel-green` 边框 + `pixel-green-bright` 文字 + 4px 偏移阴影 |
| 悬停 | 背景变 `pixel-green` + 文字变 `bg-primary` + 阴影缩小至 2px |
| 按下 | 阴影消失 + 整体位移 (4px, 4px)，模拟按下效果 |

关键：无 `transition`，状态切换瞬变。

### 4.2 卡片 `.sci-card`

- 背景：`--color-bg-card`
- 边框：4px solid `--color-border-default`
- 阴影：4px 4px 0 0 `#000`
- 顶部装饰条：4px 高 `--color-pixel-green` 实色条（`::before` 伪元素）

### 4.3 文字效果

| 类名 | 效果 | 使用场景 |
|------|------|----------|
| `.neon-text` | `pixel-green-bright` 色 + 2px 实色文字阴影 | 标题、强调文字 |
| `.neon-purple` | `pixel-pink` 色 + 2px 实色文字阴影 | 次要强调 |

### 4.4 边框效果 `.neon-border`

- 边框：4px solid `--color-pixel-green`
- 阴影：4px 4px 0 0 `#000`
- 悬停：追加 0 0 0 2px `pixel-green-bright` 外发光

## 五、动画规范

所有动画使用**步进式**（`steps()`）而非平滑过渡，保持像素风格的低帧率感。

### 5.1 核心动画

| 动画名 | 用途 | 时长 | 类型 |
|--------|------|------|------|
| `neon-pulse` | 按钮/元素脉冲 | 2s | `steps(2) infinite` |
| `text-glow` | 标题文字闪烁 | 2s | `steps(2) infinite` |
| `glitch` | 故障抖动 | 按需 | 平滑 |
| `glitch-skew` | 故障倾斜 | 按需 | 平滑 |
| `shake` | 错误抖动 | 0.5s | 平滑，6px 偏移 |
| `blink` | 光标闪烁 | 1s | `steps(1) infinite` |
| `success-pulse` | 成功脉冲环 | 按需 | 平滑 |
| `score-pop` | 分数弹出 | 按需 | 平滑，上浮 64px |
| `door-open` | 门解锁 | 按需 | `scaleY` 收缩至 0 |
| `fade-in-up` | 淡入上浮 | 按需 | 平滑 |
| `fade-in-scale` | 淡入缩放 | 按需 | 平滑 |

### 5.2 背景动画

| 动画名 | 元素 | 类名 | 时长 |
|--------|------|------|------|
| `twinkle` | 闪烁星星 | `.pixel-star` | 2~5s `steps(2)` |
| `drift-up` | 上浮像素块 | `.pixel-drift` | 10~26s 线性 |
| `drift-down` | 下落像素雨 | `.pixel-rain` | 6~12s 线性 |
| `circuit-pulse` | 电路脉冲线 | `.circuit-line` | 4s 线性 |

### 5.3 覆盖层

`.scanline-overlay` — CRT 扫描线效果：
- 全屏固定定位，`z-index: 9999`
- `pointer-events: none` 不阻挡交互
- 2px 间隔的半透明黑色横线

## 六、背景粒子系统

背景由 4 层动态元素组成，均使用**确定性伪随机位置**（避免 React 重渲染时闪烁）：

| 层级 | 数量 | 元素 | 颜色池 |
|------|------|------|--------|
| 星星 | 60 个 | 2/4/6px 方块 | green-bright, green, gold-bright, text-primary, cyan-bright |
| 漂浮块 | 12 个 | 4/6/8px 方块 | green, gold, cyan, green-bright, purple |
| 像素雨 | 15 个 | 4px 方块 | green, cyan, green-bright |
| 电路线 | 7 条 | 2px 线段 | green 渐变 |

位置公式：`(index * prime + offset) % range`

## 七、样式方案

### 7.1 分层策略

| 层级 | 方案 | 用途 |
|------|------|------|
| `index.css` `@theme {}` | Tailwind CSS 4 设计令牌 | 颜色、字体、全局变量 |
| `index.css` 工具类 | CSS class | 通用动画类（`.btn-neon`、`.sci-card`、`.pixel-star` 等） |
| `views/` 页面组件 | styled-components | 页面级布局和私有样式封装 |
| `components/` 通用组件 | styled-components | 可复用组件的样式封装 |

### 7.2 styled-components 用法规范

```tsx
import styled from 'styled-components'

// 使用 CSS 变量引用设计令牌
const Title = styled.h1`
  font-family: var(--font-display);
  color: var(--color-pixel-green-bright);
  text-shadow: 2px 2px 0px #000000;
`

// 临时属性用 transient props ($前缀)
const PixelBlock = styled.div<{ $color: string; $size: string }>`
  width: ${p => p.$size};
  height: ${p => p.$size};
  background: ${p => p.$color};
`

// 响应式用 @media
const Title = styled.h1`
  font-size: 1.875rem;
  @media (min-width: 768px) {
    font-size: 3rem;
  }
`
```

### 7.3 路由结构

路由配置在 `App.tsx` 中，页面组件放在 `src/views/` 目录：

```tsx
// App.tsx
import { BrowserRouter, Routes, Route } from 'react-router'

<BrowserRouter>
  <Routes>
    <Route path="/" element={<StartScreen />} />
    <Route path="/game" element={<GameScreen />} />
    <Route path="/victory" element={<VictoryScreen />} />
  </Routes>
</BrowserRouter>
```

| 路径 | 组件 | 说明 |
|------|------|------|
| `/` | `views/StartScreen.tsx` | 首页/开始界面 |
| `/game` | `views/GameScreen.tsx` | 游戏主界面（密室探索） |
| `/victory` | `views/VictoryScreen.tsx` | 通关界面 |

## 九、与原设计的差异

| 维度 | 原设计（赛博朋克） | 当前实现（像素复古） |
|------|---------------------|----------------------|
| 风格 | 霓虹发光 + 玻璃拟态 | 8-bit 像素 + CRT 扫描线 |
| 阴影 | 模糊发光 `box-shadow` | 实色偏移 `4px 4px 0 0 #000` |
| 字体 | 无衬线体 | Press Start 2P / VT323 |
| 动画 | 平滑过渡 | 步进式 `steps()` |
| 背景 | 浮动光点粒子 | 像素方块 + 电路线 |
| 字体渲染 | 正常平滑 | 禁止平滑 `none` |
| 色彩 | #00f0ff 霓虹青 | #41a6f6 像素蓝绿 |
