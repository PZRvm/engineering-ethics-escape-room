- 信息安全的定义与三大要素（CIA三要素）

    信息安全是指保护信息系统的硬件、软件及其相关数据，使之不因偶然或恶意的原因遭到破坏、更改、泄露。信息安全的三大核心要素是：机密性（Confidentiality，确保信息只被授权者访问）、完整性（Integrity，确保信息不被非法篡改）、可用性（Availability，确保授权用户可以随时获取信息）。后来又扩展出不可抵赖性（Non-repudiation）和可追溯性（Accountability）等要素。

- 常见信息安全威胁类型

    1. 恶意软件（Malware）：包括病毒、蠕虫、木马、勒索软件、间谍软件等，是信息安全的头号威胁。
    2. 网络钓鱼（Phishing）：通过伪装成可信实体（如银行、学校）的电子邮件或网站，诱骗用户泄露密码、银行卡号等敏感信息。
    3. 社会工程学攻击（Social Engineering）：利用人类心理弱点（如信任、恐惧、好奇心）来获取信息或访问权限，例如冒充IT部门打电话索要密码。
    4. 拒绝服务攻击（DoS/DDoS）：通过大量请求淹没目标服务器，使正常用户无法访问服务。2016年Mirai僵尸网络攻击导致美国东海岸大面积断网。
    5. 中间人攻击（Man-in-the-Middle）：攻击者在通信双方之间截获和篡改信息。
    6. 零日漏洞（Zero-day Vulnerability）：软件中尚未被发现或修补的安全漏洞，被攻击者率先利用。

- 个人信息保护法（PIPL）核心要点

    《中华人民共和国个人信息保护法》于2021年11月1日正式施行，是中国第一部全面的数据保护法律。核心要点包括：
    1. 个人信息的定义：以电子或者其他方式记录的与已识别或者可识别的自然人有关的各种信息，不包括匿名化处理后的信息。
    2. 处理个人信息的合法基础：需取得个人同意，或为订立/履行合同所必需，或为履行法定职责所必需等。
    3. 个人权利：知情权、决定权、查阅复制权、更正补充权、删除权（即"被遗忘权"的中国版本）。
    4. 敏感个人信息的特殊保护：包括生物识别、宗教信仰、特定身份、医疗健康、金融账户、行踪轨迹等信息，以及不满十四周岁未成年人的个人信息，处理时需取得"单独同意"。
    5. 跨境传输规则：向境外提供个人信息需满足特定条件（安全评估、标准合同、认证等）。
    6. 法律责任：违规者最高可处5000万元或上一年度营业额5%的罚款。

- 网络安全法律法规体系

    中国网络安全法律体系以"三驾马车"为核心：
    1. 《中华人民共和国网络安全法》（2017年实施）：确立了网络安全等级保护制度、关键信息基础设施保护制度、个人信息保护制度等基本制度。
    2. 《中华人民共和国数据安全法》（2021年实施）：建立了数据分类分级制度、重要数据保护制度、数据安全审查制度。
    3. 《中华人民共和国个人信息保护法》（2021年实施）：如上所述。
    此外还有《关键信息基础设施安全保护条例》《网络安全审查办法》《数据出境安全评估办法》等配套法规。

- 欧盟GDPR与中国PIPL的比较

    GDPR（通用数据保护条例）于2018年在欧盟生效，是全球最严格的数据保护法规之一。与PIPL的比较：
    1. 适用范围：GDPR采用"市场地"原则（只要处理欧盟居民数据就适用），PIPL主要适用于境内处理活动。
    2. 同意机制：两者都要求同意，但GDPR的同意标准更高（必须"自由、具体、知情、明确"）。
    3. 罚款：GDPR最高2000万欧元或全球营业额4%，PIPL为5000万元或上年度营业额5%。
    4. 数据主体权利：GDPR明确规定了"被遗忘权"（Right to Erasure），PIPL使用"删除权"表述。
    5. 跨境传输：GDPR采用"充分性认定"机制，PIPL采用安全评估、标准合同、认证等机制。

- 信息安全伦理原则

    1. 最小权限原则：只赋予用户完成工作所需的最小权限。
    2. 纵深防御原则：不依赖单一安全措施，而是构建多层防护体系。
    3. 安全与隐私的平衡：安全措施可能侵犯隐私（如监控员工行为），需要在两者间找到平衡。
    4. 责任共担原则：云环境中，安全责任由云服务商和用户共同承担。
    5. 透明性原则：数据处理行为应对用户透明，用户应知道自己的数据如何被使用。
    6. 安全漏洞的披露伦理：发现漏洞后应负责任地披露（先报告给厂商，给予修复时间），而非直接公开。

- 经典信息安全案例分析

    1. 2013年斯诺登事件：美国NSA前雇员揭露了"棱镜计划"，显示美国政府大规模监控全球互联网通信，引发全球对隐私权和政府监控的伦理讨论。
    2. 2017年WannaCry勒索病毒：利用Windows SMB协议漏洞在全球传播，影响150多个国家、30多万台计算机，英国国家医疗服务体系（NHS）受到严重冲击。
    3. 2018年Facebook-Cambridge Analytica丑闻：Cambridge Analytica公司未经授权收集了约8700万Facebook用户的个人数据，用于政治广告定向投放，影响2016年美国大选和英国脱欧公投。
    4. 2021年滴滴出行数据安全审查：因严重违法违规收集使用个人信息，被处80.26亿元罚款。

- 计算机专业人员的网络伦理责任

    ACM（美国计算机协会）职业伦理准则要求计算机专业人员：
    1. 为社会和人类福祉做贡献。
    2. 避免伤害他人。
    3. 诚实守信。
    4. 公正且不歧视。
    5. 尊重知识产权（包括尊重隐私）。
    6. 保守机密。
    中国计算机学会（CCF）也发布了相关职业道德规范。

    讨论：作为计算机专业学生，当你在实习中发现公司产品存在严重安全漏洞但管理层不予修复时，你会怎么做？这是典型的"吹哨人"（Whistleblower）伦理困境。
